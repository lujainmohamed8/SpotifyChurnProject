1. Feature Analysis

We are utilizing the Spotify User Activity Logs to track behavior patterns over a 6-month period.

Demographics

Age, Gender, Country. Used to identify regional or generational attrition trends.

Subscription

Free vs. Premium (Family/Duo). Critical for assessing tier-specific stickiness.

Engagement

Listening Time, Songs Per Day. Sharp declines here often precede churn.

Friction

Skip Rate & Ad Tolerance. High skips indicate low content relevance.

2. Project Objective

The Goal

To predict Customer Churn (User Attrition). Acquiring a new listener is 5x more expensive than retaining an existing one. Our model will identify "at-risk" users for proactive intervention (e.g., targeted playlists or offers).

Optimization Strategy: Maximize Recall to capture potential churners while balancing Precision to manage retention budget.

3. Proposed Algorithms

We propose an Ensemble Learning approach, pitting a robust bagging method against a state-of-the-art boosting method.

Primary: XGBoost

Extreme Gradient Boosting builds trees sequentially, where each new tree corrects the errors of the previous ones.

Strengths

  • High Accuracy: Uses gradient descent to minimize errors.
  • Class Imbalance: scale_pos_weight handles rare churn events well.
  • Pruning: Built-in regularization prevents overfitting.

Weaknesses

  • Complex hyperparameter tuning (Learning rate, depth).
  • Can overfit if the dataset is too small/noisy.

Baseline: Random Forest

A bagging technique that builds multiple independent decision trees in parallel and averages their results.

Strengths

  • Robustness: Handles outliers and noise better than boosting.
  • Parallelism: Trees are built simultaneously (faster training).
  • No Scaling: Requires less data preprocessing/scaling.

Weaknesses

  • Latency: Slower prediction speed in real-time applications.
  • Memory: High memory consumption for large forests.
  • Black Box: Less interpretable than single trees.

Technical Comparison

4. Research Context

A review of recent literature in streaming analytics and telecommunications churn.

Performance Benchmarks (2025)

According to "Comparative Analysis of Churn Prediction Models," XGBoost consistently achieves F1-scores in the 0.82–0.92 range. It outperforms deep neural networks on tabular data because decision trees are better suited for categorical variables (like Subscription Type) and require less data to converge.

Behavioral vs. Static Data

Research by IEEE Access highlights that behavioral features (Skip Rate, Session Length) have a 60% higher predictive power than static demographics (Age, Gender). This justifies our heavy focus on the "Engagement" and "Friction" feature sets.

Handling Imbalance

Academic consensus suggests that for Churn (where only ~10% of users leave), standard accuracy is misleading. Studies recommend using SMOTE (Synthetic Minority Over-sampling Technique) for the Random Forest baseline, while utilizing Weighted Loss Functions for the XGBoost model to ensure the minority class is not ignored.
